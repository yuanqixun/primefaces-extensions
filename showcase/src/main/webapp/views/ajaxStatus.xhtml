Real JSF web applications consist of various ajaxifed components. The good practise is to tell users that they should be waiting for something to finish. A global Ajax Status Indicator is very handy in ajaxifed applications. It informs users about any ajax interactions. Extended Ajax Status is similar to PrimeFaces' one, but it also considers JSF standard ajax requests in addition to PrimeFaces ajax requests. You can use a mix composed of standard f:ajax with any standard and not standard components + ajaxified components from PrimeFaces. The implementation is smart enough and avoid collisions with parallel running ajax requests. Furthermore, the cursor is set to "wait" if an ajax request is running. It's how desktop applications behaves.
